ProjX User Guide

GitHub Repository: https://github.com/pcr1304/sw_project

ProjX is a domain-specific language for modeling projectile motion, running physics simulations, comparing trajectories, and generating interactive browser-based visualizations. With a compact .px syntax, it eliminates the need to stitch together separate calculation, plotting, and presentation tools.

Target Audiences & Use Cases

  • Students and instructors demonstrating kinematics and projectile motion.
  • Developers prototyping physics experiments and bounce scenarios.
  • Anyone looking to compare identical launches across different gravities visually.

1. Setup & Build

Prerequisites

  • OCaml 5.x
  • opam (OCaml package manager)
  • dune (build system)
  • yojson (JSON generation)

Installation

  1. Install OCaml and opam. Then install missing dependencies: bash opam install dune yojson
  2. Build the project: bash dune build
  3. Run the tests (testing lexer/parser integration): bash dune test

2. Syntax Basics

A .px script consists of statements grouped via blocks { ... }. Semicolons are not used. Variables and projectiles must be declared before use.

Comments

Comments use the # character. Anything trailing # is ignored.

# This is a comment
let g = 9.8

Variables & Expressions

Declare and update numeric variables. You can intermix expressions and parentheses.

let v0 = 25 + 5
set v0 = v0 * 2

3. Projectile & Simulation Blocks

Projectile Block

Define a projectile setup. angle and speed are strictly required.

projectile cannon {
  angle 45               # Required (degrees)
  angle_azimuth 30       # Optional horizontal aim (degrees)
  speed 70               # Required
  launch_from (20, 30, 40) # Optional start pos (x, y, z)
  mass 5.0               # Optional (for drag calculation)
  drag_coefficient 0.3   # Optional (for drag calculation)
  cross_section 0.05     # Optional (for drag calculation)
}

Simulate Block

A simulate block executes a physics scenario and generates the plot. Rule: It must contain exactly one gravity and at least one plot statement.

simulate {
  gravity 9.8
  air_resistance true
  air_density 1.225
  wind_y 4.0      # Optional X/Y/Z wind influence

  plot cannon     # Draw the trajectory

  # Query Metrics
  range cannon
  max_height cannon
  max_range cannon
  max_rectangle cannon

  # Dynamic checks
  check range.cannon() > 200
  collide cannon sweep
}

4. Dot Queries

Dot queries allow fetching derived values dynamically in conditional checks or variable assignments.

  • range.ball(): Range of the projectile
  • max_height.ball(): Max height achieved
  • max_range.ball(): Theoretical max range
  • max_rectangle.ball(): Maximum rectangle area beneath the arc
  • min_vel.ball(x, h): Minimum velocity needed to clear a target tower
  • collide.(p1, p2): Returns 1 if collision occurs, 0 otherwise
  • min_dist.(p1, p2): Reports the minimum distance between two arcs

Example in context:

let r = range.cannon()
if range.cannon() > 100 and max_height.cannon() > 20 {
  let success = 1
}

5. Advanced Features

Game Mode

Swap a standard simulation for an interactive web game:

game {
  planet earth  # Valid: earth, moon, mars, jupiter, sun
  level 1
  lives 3
}

Comparing Scenarios (Forking)

fork creates comparative simulation branches for an existing projectile. Each branch is effectively a simulation block requiring its own gravity and plot.

fork cannon {
  branch "Earth" {
    gravity 9.8
    plot cannon
  }
  branch "Moon" {
    gravity 1.6
    plot cannon
  }
}

Control Flow

Loops (for, repeat, while) and conditionals (if/else) are built-in.

for a from 15 to 75 step 15 {
  projectile sweep { angle a  speed 70 }
  simulate { gravity 9.8  plot sweep }
}

6. Execution & the Compilation Pipeline

When a .px program executes, it follows a systematic pipeline from raw input to generated graphical view:

flowchart TD
    A([.px Source File]) --> B(Tokenizer: lib/tokenizer.ml)
    B --> C(Parser: lib/parser.ml)
    C --> D(Semantic Checker: lib/checker.ml)
    D --> E(Evaluator & Physics: lib/eval.ml & lib/physics.ml)
    E --> F(JSON Emitter: lib/json_emit.ml)
    F --> G(HTML Generator: bin/main.ml)
    G --> H([HTML Interactive UI])
  • Tokenizer (lib/tokenizer.ml): Converts raw text strings into distinct linguistic tokens.
  • Parser (lib/parser.ml): Composes the sequences of tokens into an Abstract Syntax Tree (AST).
  • Semantic Checker (lib/checker.ml): Validates variable scopes, projectile dependencies, and simulation rules.
  • Evaluator & Physics (lib/eval.ml, lib/physics.ml): Evaluates queries, calculates kinematics, and processes interactions.
  • JSON Emitter (lib/json_emit.ml): Bundles output metrics and positional data into a structured format for rendering.
  • HTML Generator (bin/main.ml): Automates rendering all JSON properties neatly into a standalone interactive web application.

Run your .px file:

dune exec projx -- input/queries.px

Outputs JSON to the console and generates an interactive web interface (e.g., input/queries.html). Open the .html file inside any web browser for visualization.
